Conditions and Predicates

In a rule, each when and test section contains a condition. A condition specifies a logical assertion about an element. A condition gives a result that is either true or false.

In a when section, a true condition in means that a rule should be applied to the element. If all conditions in the when sections of a rule are false, then does not apply the rule.

In a test section, a true condition means that the matched element meets the rule’s criteria. When it finds a test condition that gives true, then considers the matched element to correctly follow the rule. If all single test conditions are false, then considers the element to have broken the rule and adds the rule’s message to a report.

You compose conditions in terms of the properties of an XML element, such as its tag, attributes, the text it contains, and the elements that surround it. The following example matches table elements:

when: self table

The text self table is the condition. The word self is a predicate in the condition. A predicate specifies the property of an element to check. In this case, the self predicate checks the element’s tag. If an element’s tag is table, then this predicate is true. Therefore, the condition is true. Other elements, like p or h1, cause this condition to give false.

Tip: you can view a Flare file’s XML elements in the Flare internal text editor or by opening a Flare file in an external text editor, like Notepad.

Some predicates, like self, need a tag, attribute, or number. Others require none, like fail. And some predicates need text, which must be enclosed in straight quotation marks ("). For example, the following condition is true when the text of an element has the word cat followed by a pound sign:

when: contains "cat#"

You can make a condition more precise by combining predicates with the word and. The following example tests that a an unordered list (ul) is in a text box (div element):

test: self ul and parent div

You can reverse a predicate’s result, making a true predicate false and vice versa, with the word not. For example, the following matches li elements and tests that they do not contain p elements:

when: self li
test: not child p

The not word applies only to the predicate that it precedes, not the whole condition. For example, these test sections give the same result, which is to test that an element does not contain a paragraph and is in a table:

test: not child p and ancestor table
test: ancestor table and not child p
