What’s New

New features and bug fixes:

Version Features and fixes

Feature: Completely new way for defining rules. There is no need for you to learn Python programming to customize rules. You can create and modify rules using a simple, straightforward language. For more information, see Creating and Customizing Rules.

Feature: You can see the file name of the rule from which a message was issued by hovering over the word “error” or “warning” in a report. This is useful when developing and troubleshooting rules.

Fix: FlareLint.bat correctly accepts paths with space characters.

Limitation: in Microsoft Windows 10, you can’t drag and drop Flare project files onto the FlareLint.bat. Instead, copy FlareLint.bat into each of your Flare project folders. For more information, see Installing and Using.

Feature: New predicate: pov-first.

Fix: loads rules that you created or edited in Microsoft Notepad. Notepad quietly inserts an invisible Byte order mark (BOM) character to the beginning of text files. ignores the BOM instead of reporting an error in the rule file.

1.2

Improvements to this manual.

In the HTML report, file names appear with their entire path. You can click the path to open the offending file in a new browser tab or window. You can copy and paste this path to edit the file directly in your favorite editor.

FlareLint.bat prompts you to press any key only when there is an error in invoking . Removed the -q command-line argument.

You can inspect, add, and modify rules without mangling your Python installation or installation. For more information, see Managing Rules and See "Modifying and Creating New Rules".

Rules may check for language. The flarenode.Node.lang() method returns True if its argument matches the xml:lang attribute of the node, its nearest ancestor, or the Flare project, in that order. For details, see the flarenode.py module.

Rules are more easily customizable. For details, see the rule source code.

Other small improvements to the source code.
